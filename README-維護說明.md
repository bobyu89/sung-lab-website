## 1. 最新消息／活動：建立與發布 Google 試算表

網站的「最新消息」與活動內容，是從一份 **Google 試算表** 自動讀取的。步驟如下：

### 1.1 建立試算表

1. 到 [Google 試算表](https://sheets.google.com) 新增一份空白試算表，取名例如「護理創新研究室網站內容」。
2. 在試算表下方建立**一個工作表分頁**，分頁名稱必須**完全是英文小寫**：
   - 命名為 `news`（最新消息）

> 目前網站只需要 `news` 這一個工作表分頁。未來若要新增首頁「精選活動」功能，屆時會再另外新增 `events` 工作表分頁並提供對應說明。

### 1.2 欄位標題（非常重要：必須逐字比對，大小寫、英文都不能改）

網站程式是靠「欄位標題文字」去對應資料的，所以標題文字必須跟下面**一模一樣**（英文小寫，不要有空格）：

**`news` 分頁，第一列（標題列）依序填入：**

| date | category | content | link |
|---|---|---|---|

- `date`：日期，建議格式 `YYYY-MM-DD`，例如 `2026-07-06`
- `category`：分類，請填以下四選一（純文字）：`榮譽`、`發表`、`活動`、`公告`
- `content`：這則消息的內容文字
- `link`：相關連結網址（沒有可留空白）

> **提醒**：欄位標題如果打錯字、多打空格、或大小寫不同（例如打成 `Date` 或 `日期`），網站會讀不到資料，屆時會自動顯示預設的示範內容，不會出現錯誤畫面，但新內容不會顯示，請務必仔細核對。

### 1.3 發布試算表（讓網站讀得到）

1. 在試算表右上角點「**共用**」。
2. 選擇「知道連結的任何人」都可以「檢視」（不需要能編輯，網站只讀取不寫入）。
3. 記下瀏覽器網址列中試算表的 ID，網址長得像：
   `https://docs.google.com/spreadsheets/d/【這一長串英數字就是SHEET_ID】/edit`
4. 把這串 ID 交給工程人員（或依照第 3 節自行貼到 `js/data.js`）。

之後每次要新增「最新消息」，只要**直接在 `news` 分頁多加一列資料**即可，網站會自動抓到最新的內容，不需要重新部署網站。

## 2. 活動照片：Google 雲端硬碟資料夾與 API 金鑰

「合作交流」頁面的活動照片牆，是從一個 **Google 雲端硬碟資料夾** 自動抓圖的。

### 2.1 建立雲端硬碟資料夾

1. 到 [Google 雲端硬碟](https://drive.google.com) 新增一個資料夾，例如「網站活動照片」。
2. 把資料夾設為「知道連結的任何人」可以「檢視」（右鍵 → 共用 → 一般存取權設為「知道連結的任何人」）。
3. 網址列會顯示資料夾 ID：
   `https://drive.google.com/drive/folders/【這一長串就是 DRIVE_FOLDER_ID】`

### 2.2 申請 Google API 金鑰

這一步需要 Google 帳號，操作稍微技術性，若不熟悉建議請工程人員協助，步驟概念如下：

1. 到 [Google Cloud Console](https://console.cloud.google.com/) 建立一個新專案（或使用現有專案）。
2. 在「API 和服務」中啟用 **Google Drive API**。
3. 在「憑證」頁面建立一組「API 金鑰」（API Key）。
4. **重要（安全性）**：建立金鑰後，務必點選「限制金鑰」，將「應用程式限制」設為「HTTP 參照網址（網站）」，只允許您網站的網域（例如 `https://您的網站.web.app/*`）能使用這把金鑰，避免被其他網站盜用造成額度被濫用或產生費用。
5. 複製這把金鑰字串，交給工程人員或依第 3 節自行貼上。

### 2.3 上傳照片

之後只要把照片直接拖曳上傳到這個雲端硬碟資料夾，網站的照片牆就會自動顯示（依照第 4 節的命名規則命名，照片說明文字會取自檔名）。

## 3. 把試算表與雲端硬碟接上網站（CONFIG 設定）

拿到上面的三組資訊後（試算表 ID、雲端資料夾 ID、API 金鑰），請工程人員（或您自己用記事本）打開檔案：

```
js/data.js
```

找到最上面的這一段（大約第 21–25 行）：

```js
const CONFIG = {
  SHEET_ID: "",
  DRIVE_FOLDER_ID: "",
  DRIVE_API_KEY: ""
};
```

把三個空的引號 `""` 之間，貼上對應的 ID／金鑰即可，例如：

```js
const CONFIG = {
  SHEET_ID: "1AbCdEfGhIjKlMnOpQrStUvWxYz1234567890",
  DRIVE_FOLDER_ID: "1XyZ9876543210AbCdEfGhIjKlMnOp",
  DRIVE_API_KEY: "AIzaSyABCDEFG1234567890hijklmnop"
};
```

存檔後重新部署網站（見第 6 節），網站就會改成讀取真實的試算表與照片，不再顯示預設示範內容。**在填入這三項之前，網站會自動顯示內建的示範內容，不會出現空白或錯誤畫面**，所以可以不急著設定，準備好再處理即可。

## 4. 照片命名規則

上傳到雲端硬碟資料夾的照片，建議依照以下格式命名，網站會**自動把檔名（去掉副檔名）當作照片說明文字**顯示：

```
2026-06-15 活動名稱.jpg
```

例如：

- `2026-06-15 護理創新研討會.jpg` → 網站顯示說明文字為「2026-06-15 護理創新研討會」
- `2026-03-20 3D列印教具工作坊.png` → 顯示「2026-03-20 3D列印教具工作坊」

建議：日期放最前面方便管理，檔名中請避免使用特殊符號（如 `/ \ : * ? " < > |`），中文、英文、數字、空格、底線都可以使用。

## 5. 如何修改網站名稱、統計數字

網站名稱、老師姓名、聯絡方式、統計數字（發表著作篇數、研究計畫數等）都集中寫在同一個地方，方便之後修改：

打開檔案 `js/data.js`，最上方（約第 6–15 行）：

```js
const SITE = {
  nameZh: "護理創新及專科訓練研究室",
  nameEn: "Nursing Innovation & NP Training Lab",
  tagline: "結合專師照護、實證研究、教育創新與科技應用",
  pi: "宋建美 助理教授",
  dept: "國防醫學大學護理學院",
  email: "sungcm@mail.ndmutsgh.edu.tw",
  phone: "02-87923100 分機 18765",
  stats: { publications: 23, projects: 11, areas: 4, members: null }
};
```

- 要改研究室名稱：改 `nameZh`（中文）、`nameEn`（英文）
- 要改標語：改 `tagline`
- 要改老師稱謂：改 `pi`
- 要改系所單位：改 `dept`
- 要改 Email／電話：改 `email`／`phone`
- 要改統計數字：改 `stats` 裡的數字，例如發表著作篇數增加了，就把 `publications: 23` 改成新的數字
  - `members` 目前是 `null`，代表首頁會顯示「招募中」；等確定團隊人數後，把 `null` 改成實際數字（例如 `members: 5`）即可改顯示成「5」

改完存檔後，重新部署網站（見第 6 節）即可看到更新。

## 6. 如何把網站發布到正式網址（Firebase）

網站目前是純靜態網頁（不需要資料庫、不需要後端伺服器），最簡單的免費發布方式是使用 **Firebase Hosting**。

### 6.1 前置需求（僅需設定一次）

1. 需要一台已安裝 [Node.js](https://nodejs.org/) 的電腦（工程人員通常已具備）。
2. 安裝 Firebase 工具：在終端機執行
   ```
   npm install -g firebase-tools
   ```
3. 登入 Google 帳號：
   ```
   firebase login
   ```

### 6.2 建立 Firebase 專案

1. 到 [Firebase 主控台](https://console.firebase.google.com/) 點「新增專案」，依畫面指示建立一個新專案（例如命名 `sung-nursing-lab`）。
2. 在專案內開啟「Hosting」功能。

### 6.3 部署網站

專案資料夾內已經有設定好的 `firebase.json`，所以工程人員只需要在專案資料夾內執行：

```
firebase deploy
```

> **提醒（第一次部署才需要）**：目前專案資料夾裡還沒有 `.firebaserc` 這個檔案（用來記住「這個專案要部署到哪一個 Firebase 專案」）。所以**第一次**部署前，工程人員需要先執行 `firebase login` 登入，再執行 `firebase use --add`，畫面會列出您在 6.2 建立好的 Firebase 專案，選擇（或視需要建立）對應的專案一次即可，之後每次 `firebase deploy` 就會自動記得部署到同一個專案，不用重複這個步驟。

跑完之後終端機會顯示一個網址（類似 `https://sung-nursing-lab.web.app`），這就是網站的正式公開網址，之後可以請學校 IT 或網域商將 `.edu.tw` 網域指向這個地址（如需自訂網域，Firebase 主控台的「Hosting」頁面有「新增自訂網域」的引導步驟）。

**之後每次更新內容**（改了 `js/data.js` 或任何檔案），只要重新執行一次 `firebase deploy` 就會發布最新版本。

## 7. 已知限制

以下是目前網站已知、尚待後續處理的事項，記錄於此方便追蹤：

1. **Lighthouse 效能與無障礙分數尚未跑過官方檢測**：本次驗收環境沒有安裝 Node.js／Chrome 自動化工具鏈，因此無法執行 `npx lighthouse` 產生正式分數報告。已改用手動方式檢查（頁面檔案大小、無障礙檢查清單），結果詳見驗收報告，**建議之後在有 Node.js 環境的電腦上執行一次 Lighthouse，目標效能與無障礙分數皆 ≥ 90**，指令為：
   ```
   npx --yes lighthouse http://localhost:5175 --only-categories=performance,accessibility --chrome-flags="--headless" --output=json --output-path=./lighthouse.json
   ```
2. **首頁大圖 `assets/hero-illustration.png` 尚未提供**：目前會自動顯示粉橘漸層愛心圖示佔位，不影響版面，正式插畫圖檔規格請見 `assets/素材規格清單.md`。
3. **「團隊成員」頁面除主持人外為佔位內容**：等老師確認完整團隊成員名單（含照片）後再行補上。
4. **「教學資源」頁面部分區塊（課程教學／學生作品／臨床教學）為「建置中」佔位**：待相關內容確認後補上。
5. **「開放資源」頁面（開放工具／衛教素材／研究 Protocol）為「建置中」佔位**：待相關內容確認後補上。
6. **論文篇數顯示 23，目前逐字收錄 21 篇完整書目（20 英文＋1 中文）**：來源頁面標示共 22 篇英文＋1 篇中文，尚有 1–2 篇英文書目待補齊完整引用資料，目前頁尾已加註「其餘書目補齊中」。
7. **少數小型文字（如首頁「實證為本」等膠囊標籤 chip）對比度略低於 WCAG AA 標準**：實測約 3.6–3.7:1（標準文字需 4.5:1），屬於全站共用的設計樣式，建議日後請設計/工程人員評估是否加深顏色或加大字級以符合無障礙標準，本次未逕行修改以免影響其他既有頁面的視覺一致性。
